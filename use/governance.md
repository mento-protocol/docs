# Governance

Mento is governed by token holders and the community. This page covers participating, tokenomics, components, and watchdogs at a high level.

---

## Participating

- **Hold and stake:** MENTO token holders can stake (e.g. veMENTO) to get voting power.
- **Vote:** Proposals (parameter changes, upgrades, treasury) are voted on by the community. Use the governance UI to vote.
- **Propose:** Eligible participants can create proposals; follow the governance process and templates.

## Tokenomics

- **MENTO** is the governance and utility token. Use cases: governance (voting, proposals), incentives (e.g. LP or rebalance incentives), and possibly other protocol roles as defined by governance.
- **veMENTO** (or equivalent): locked MENTO for increased voting power; lock duration may affect weight. See the governance docs for current mechanics.

## Governance components

- **Governance contract(s):** Execute approved proposals (parameter updates, upgrades). See build docs for addresses.
- **Treasury:** Protocol-owned assets; usage is governance-controlled.
- **Parameter control:** Fee, rebalance incentive, trading limits, and other pool parameters can be updated via governance.

## Watchdogs and safety

- **Watchdogs:** Roles or mechanisms that monitor protocol state (e.g. oracle, limits, insolvency) and can pause or escalate. Described in the security and governance docs.
- **Pause / circuit breakers:** In emergencies, governance or watchdogs may pause trading or rebalancing. See [Limits & breakers](../concepts/limits-and-breakers.md) and [Security](security.md).
